## 九、给你的最后提醒

1. **不要凭印象写**：每个数字、每个命令、每个路径都先用 `cat`/`grep`/`ls` 验证当前文件状态。
2. **不要复制 Ibex 文档原文**：参考其结构和深度，**用 EH2 自己的代码和实测数据**重写。
3. **平台状态以 git HEAD + 工作区为准**：当文档与当前代码冲突时，**改文档不改代码**（代码刚经过严格验证）。
4. **`.progress.md` 是契约**：你每写完一个 .rst 必须立即更新一行，禁止积压。
5. **遇到不确定的设计意图，去读 `docs/adr/` 和 `docs/release-notes-v1.1.md`**，不要猜。
6. **预计总工作量**：~14 个阶段 × ~6-12 小时 = ~100-150 小时，分多次会话完成。每次会话开始先读 `.progress.md` 接上次进度。
