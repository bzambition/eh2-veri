## Architecture

```
core_eh2_tb_top (testbench)
  |
  +-- eh2_veer_wrapper (DUT)
  |     +-- AXI4 ports: LSU, IFU, SB, DMA
  |     +-- Trace ports: trace_rv_i_*
  |     +-- IRQ ports: extintsrc_req, timer_int, soft_int, nmi_int
  |     +-- JTAG ports: jtag_tck, jtag_tms, jtag_tdi, jtag_tdo, jtag_trst_n
  |
  +-- axi4_slave_mem x3 (LSU, IFU, SB behavioral memory)
  +-- clk_rst_if (clock and reset)
  +-- eh2_trace_intf (trace signals)
  +-- eh2_dut_probe_intf (internal DUT probes)
  +-- eh2_irq_intf (interrupt signals)
  +-- eh2_jtag_intf (JTAG signals)
  |
  +-- core_eh2_env (UVM environment)
        +-- lsu_agent / ifu_agent / sb_agent (AXI4 passive agents)
        +-- irq_agent (interrupt active agent)
        +-- jtag_agent (JTAG active agent)
        +-- trace_monitor (instruction commit monitor)
        +-- dut_probe_monitor (register writeback monitor)
        +-- cosim_scoreboard (DUT vs reference model comparison)
```

## Key Design Decisions

### 1. Bus Interface: AXI4
- EH2 uses AXI4 (not simple req/gnt like Ibex)
- Parameterized AXI4 agent with 4 ports (IFU/LSU/SB/DMA)
- LSU and IFU are 64-bit data width

### 2. Memory Architecture: Tightly Coupled
- DCCM (0xF0040000) and ICCM (0xEE000000) are internal to DUT
- External memory handled by AXI4 slave memory models
- Test loading via backdoor hierarchical access

### 3. Co-simulation: DPI-based
- Abstract Cosim C++ class with step(), notify_reg_write(), notify_mem_access()
- DPI-C bridge for SystemVerilog integration
- Scoreboard compares DUT trace with reference model

### 4. Debug: JTAG/DMI
- Full JTAG agent implementing DMI protocol
- Supports halt/resume, abstract commands
- TAP state machine in driver

### 5. Coverage: Bind-based
- SystemVerilog `bind` attaches coverage interface to DUT
- Coverpoints for: instruction categories, stalls, branches, exceptions,
  interrupts, debug mode, dual-issue, cache events, LSU external accesses
- Cross coverage for key interactions

### 6. Regression: Python-driven
- Python scripts orchestrate: generate -> compile -> simulate -> check -> report
- YAML testlist defines all tests with options
- JUnit XML for CI integration
- Parallel execution support

## EH2-Specific Features Verified

| Feature | Coverage Target | Method |
|---------|----------------|--------|
| RV32IMAC ISA | All instruction categories | riscv-dv random + directed |
| Zba/Zbb/Zbc/Zbs | All bitmanip instructions | eh2_bitmanip_stream |
| Dual-issue (I0+I1) | Both pipes active | cp_dual_issue cross |
| PIC interrupts | 127 sources, priority | eh2_irq_agent + eh2_pic_int_stream |
| JTAG debug | Halt/resume, CSR access | eh2_jtag_agent |
| Custom CSRs | All WD CSRs | eh2_csr_access_stream |
| Pipeline stalls | Load/store/decode/fetch | Stall coverpoints |
| Branch prediction | Taken/not-taken, mispredict | Branch cross coverage |
| Cache behavior | Hit/miss, external access | Icache/LSU coverpoints |
| LR/SC atomics | Reserve/conditional store | eh2_atomic_stream |

## Usage Examples

```bash
# Compile testbench
make compile SIMULATOR=vcs

# Run single test
make run TEST=riscv_arithmetic_basic_test SEED=42

# Run smoke test
make smoke

# Run nightly regression
make nightly

# Run weekly regression (more iterations)
make weekly

# Run sign-off gates
make signoff_quick
make signoff SIGNOFF_PROFILE=full PARALLEL=4
python3 dv/uvm/core_eh2/scripts/signoff.py --profile full --dry-run

# Generate riscv-dv instructions
make gen TEST=riscv_random_instr_test SEED=1

# Run regression with Python script directly
python3 dv/uvm/core_eh2/scripts/run_regress.py \
  --testlist dv/uvm/core_eh2/riscv_dv_extension/testlist.yaml \
  --simulator vcs --iterations 5 --parallel 4

# Collect coverage
make cov SIMULATOR=vcs
```

## Issues and Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| No standard RVFI on EH2 | HIGH | Use DUT probing for register writeback |
| AXI4 complexity | MEDIUM | Parameterized agent, gradual feature add |
| QEMU DPI integration | MEDIUM | Start with offline comparison |
| Dual-thread support | LOW | Default to single-thread initially |
| Bind hierarchy path | MEDIUM | May need adjustment per EH2 config |

## Framework Completion Status

All framework components are implemented. The following are independent
follow-up tasks (not part of the framework):

1. **Compilation verification** - Run `make compile SIMULATOR=vcs` to validate filelists
2. **Smoke test** - Execute `make smoke` to validate basic flow
3. **Cosim tuning** - Tune Spike ISS integration for EH2-specific CSRs
4. **Coverage refinement** - Populate waiver files after first coverage run
5. **Directed tests** - Add more tests for edge cases (dual-issue, bitmanip, PMP)
6. **CI/CD integration** - Wire up to continuous integration pipeline
